OWASP Demo-Lab

Modul 183 — Java Spring Security. Jede Schwachstelle wird zuerst vorgeführt und direkt daneben mit der Gegenmassnahme abgesichert.

Lern-Sandbox: Alle Daten liegen in einer flüchtigen In-Memory-Datenbank (reine Demo-Daten, keine echten Personendaten). Die „verwundbaren" Pfade wirken ausschliesslich auf diese Sandbox.

Demonstrierte Risiken

A01

Broken Access Control

IDOR in der Bibliotheksverwaltung — fremde Ausleihen über die User-ID abrufen, abgesichert per Ownership-Check und @PreAuthorize.

Demo öffnen
A05

Injection

SQL-Injection in der Produktsuche durch String-Verkettung — abgesichert mit Parameterbindung (Spring Data JPA).

Demo öffnen
A04

Cryptographic Failures

Passwortspeicherung Klartext / MD5 gegenüber BCrypt mit Salt.

Demo öffnen
A02

Security Misconfiguration

Offene Actuator-Endpunkte und fehlende Security-Header — abgesichert über Exposure-Limit und CSP/HSTS/X-Frame-Options.

Demo öffnen
A07

Authentication Failures

Brute-Force/Credential-Stuffing — abgesichert mit Account-Lockout, IP-Rate-Limit und CAPTCHA. Direkt am Login erlebbar.

Zum Login
A10

Mishandling of Exceptions

Stack-Traces im HTTP-Response gegenüber generischer Fehlerseite via @ControllerAdvice.
